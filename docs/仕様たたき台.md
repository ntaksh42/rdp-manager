# RDP マネージャ（仮称: RdpManager）仕様たたき台

> ステータス: **ドラフト v0.1（議論用たたき台）**
> 最終更新: 2026-06-22
> スタック: .NET 8 / WPF（Windows ネイティブ）
> コンセプト: **「自分専用 RDCMan」— 接続先をツリーで整理し、タブで即接続。規模は数台〜10台。**

---

## 1. 背景・目的

社内サーバー等への RDP 接続を日常的に行う中で、接続先を整理し・素早く接続し直せるツールが欲しい。既存の定番（RDCMan / mRemoteNG / Royal TS / Devolutions RDM）はそれぞれ強力だが、

- RDCMan: 無償・安定だがクローズドソースで機能追加できず、UI が古い／リサイズに再接続が必要などの制約
- mRemoteNG: OSS・多機能だが WinForms ベースで重く、設定が複雑
- Royal TS / Devolutions RDM: 高機能だが商用・オーバースペック

という状況。**「自分の業務に必要な最小限を、軽量・モダンな WPF で、自分でメンテできる形で持つ」**ことを目的とする。多人数チーム共有・PAM・監査といった企業向け機能は狙わない。

---

## 2. スコープ

| 区分 | 内容 |
|---|---|
| **対象規模** | 接続先 数台〜10台程度（個人〜小チーム） |
| **対象プロトコル** | RDP のみ（MVP）。将来 SSH/VNC を検討 |
| **対象 OS** | Windows 11 / Windows Server 2016 以降（接続先・実行環境とも） |
| **配布** | 個人利用想定。単一実行ファイル or 自己完結発行（self-contained） |
| **共有モデル** | ローカル単独。クラウド Vault・サーバー連携は対象外 |

### 競合から取り込む設計ポイント
- **継承モデル**（RDCMan/mRemoteNG）: フォルダに設定を持たせ、接続は「親から継承」で簡素化。
- **資格情報プロファイル**（RDCMan の Credential Profiles）: 共通アカウントを1箇所で管理し、パスワード変更を1回で反映。
- **DPAPI 暗号化**（RDCMan）: `CryptProtectData` 相当でローカルユーザー権限に紐付け暗号化。
- **ad hoc 接続 → 正式接続へ昇格**（RDCMan の Connect To グループ）: 一時接続を後でツリーに保存。
- **キー送信メニュー**（全製品）: リモート内で RDCMan を動かす場合等に必須。

## 4. ペルソナ・主要ユースケース

**ペルソナ**: 開発者／運用担当。社内の検証・本番サーバー数台〜10台に日常的に RDP 接続する。

**ユースケース**
1. ツリーから目的のサーバーをダブルクリック → タブでセッションが開く。
2. 「検証環境」フォルダの共通アカウントを資格情報プロファイルで一元管理。パスワード変更時は1回更新。
3. 複数サーバーを同時に開き、タブで切り替えながら作業。
4. 一時的に未登録ホストへ接続（ad hoc）し、よく使うなら後でツリーに保存。
5. 全画面でじっくり作業 → ホットキーで復帰。
6. インクリメンタル検索で多数のサーバーから即座に目的の1台を絞り込む。

## 5. 機能要件

優先度: **[M]=MVP / [2]=Phase2 / [F]=将来**

### 5.1 接続ツリー・グループ
- [M] フォルダ階層で接続先をグルーピング（**自由ネスト**。RDCMan の「同種のみ」制約は設けない＝フォルダ内にフォルダと接続を混在可）。
- [M] 接続・フォルダの追加／編集／削除／複製。
- [M] ドラッグ&ドロップで移動・並べ替え。
- [M] 右クリックコンテキストメニュー（接続/編集/削除/複製/新規）。
- [M] 接続のダブルクリック＝接続。Enter キーでも接続。
- [2] グループ単位で「全接続/全切断」。
- [2] お気に入り／最近使った接続（仮想グループ）。

### 5.2 接続設定（RDP パラメータ）
接続オブジェクトが持つ設定。各項目は**「親フォルダから継承」**チェックで継承可能。

- [M] ホスト名 / IP、ポート（既定 3389）。
- [M] 表示名（任意。ツリー表示用）。
- [M] コメント／メモ。
- [M] 画面サイズ: クライアント領域に合わせる / 全画面 / 固定解像度指定。
- [M] 色深度。
- [M] スマートサイジング（ウィンドウに合わせて拡縮表示）。
  - ※ RDCMan の制約（リサイズ時に再接続が必要）に対し、**スマートサイジング ON で表示スケールのみ追従**させ UX を改善する方針（論理解像度の変更は再接続を要する点は同じ）。
- [2] 接続時に実行するプログラム（作業ディレクトリ含む）。
- [M] コンソールセッションへ接続するか。
- [F] Azure ロール負荷分散 Cookie（RDCMan 相当。当面不要）。

### 5.3 資格情報・認証
- [M] ユーザー名 / パスワード / ドメイン（`domain\user` 形式も可）。
- [M] **資格情報プロファイル**: 共通アカウントを名前付きで保存し、複数接続から参照。パスワード変更は1箇所で反映。
- [M] **継承**: 接続はプロファイル参照 or 親フォルダの資格情報を継承可能。
- [M] **DPAPI 暗号化**（`ProtectedData` / CurrentUser スコープ）でパスワードを保存。**平文保存は禁止**。
- [M] パスワード未保存時は接続時にプロンプト（任意で都度入力）。
- [2] `[server]` / `[display]` 置換（ローカル管理者ログイン用。RDCMan 相当）。
- [F] X509 証明書による暗号化（共有用途。当面不要）。
- [F] 外部パスワードマネージャ連携。

### 5.4 表示・セッション領域
- [M] **タブ表示**: 接続ごとにタブを生成、複数同時接続。
- [M] タブの閉じる／切断／再接続。
- [M] **全画面モード**: ホットキー（既定 `Ctrl+Alt+Break`、設定可）で切替。
- [M] 接続バー（全画面時の純正 UI バー）の表示/自動非表示。
- [2] **サムネイル一覧**: フォルダ選択時に配下サーバーをライブサムネイル表示。
- [2] マルチモニタ・スパン表示。
- [M] 接続状態の視覚表示（接続中/切断/接続試行中）。

### 5.5 セッション操作
- [M] 接続 / 切断 / 再接続 / ログオフ。
- [M] **キー送信**: `Ctrl+Alt+Del`、`Win` キー等の特殊キーをリモートへ送信（メニュー）。
- [2] 起動時に前回接続していたセッションの復元（プロンプト/自動）。
- [2] 再接続仮想グループ（接続成功まで自動リトライ）。

### 5.6 ローカルリソース・リダイレクト
- [M] クリップボード共有 ON/OFF。
- [M] ドライブ／プリンタのリダイレクト ON/OFF。
- [M] サウンド: ローカル再生 / リモート再生 / 無効。
- [M] Windows キー組み合わせの適用先（ローカル / リモート / 全画面時のみリモート）。
- [2] スマートカード、ポートのリダイレクト。

### 5.7 ゲートウェイ
- [M] TS Gateway サーバー設定（FQDN、認証方式、ローカルアドレスのバイパス）。
- [M] ゲートウェイ資格情報をリモート接続と共有するか。

### 5.8 検索・一括操作
- [M] **インクリメンタル検索**（ツリー上で表示名/ホスト名を絞り込み）。
- [2] 正規表現検索（`folder\server` 形式マッチ。RDCMan 相当）。
- [2] **パターン一括追加**（`server[01-10]`、`{a,b,c}` 展開。RDCMan 相当）。
- [2] テキスト/CSV からの接続インポート（1行1ホスト）。

### 5.9 ad hoc 接続
- [M] ツリー未登録のホストへその場で接続（クイック接続バー）。
- [2] ad hoc 接続を右クリックでツリーに保存（正式接続へ昇格）。

### 5.10 設定永続化・ファイル
- [M] 接続定義を **JSON ファイル**で保存（後述スキーマ）。
- [M] アプリ設定（ウィンドウ位置・サイズ・ホットキー等）を別ファイルで保存。
- [2] 接続定義ファイルのインポート/エクスポート（バックアップ・移行）。
- [F] 複数ファイル（ワークスペース）同時オープン。

## 6. 非機能要件

### 6.1 セキュリティ
- パスワードは **DPAPI（CurrentUser）** で暗号化し JSON 内に Base64 で格納。平文保存しない。
- 暗号化済みデータは**別 PC・別ユーザーでは復号不可**（DPAPI の仕様）。移行時は再入力前提（または将来 X509 対応）。
- ログ・例外メッセージに資格情報を出力しない。
- クリップボード経由のパスワード貼り付けに依存しない（純正 RDP の資格情報フィールドへ直接設定）。

### 6.2 性能
- 起動 2 秒以内（接続ツリー読み込み含む、10台規模）。
- 同時接続 5〜10 タブで操作が破綻しないこと。

### 6.3 配布・運用
- .NET 8 self-contained 発行（ランタイム同梱）も可能にし、ランタイム未導入環境でも動作。
- 単一ユーザー設定は `%APPDATA%\RdpManager\` 配下に保存。

### 6.4 アクセシビリティ・UX
- キーボード操作完結（接続・検索・タブ移動）。
- ダーク/ライトテーマ対応（Phase2）。

## 7. 技術アーキテクチャ

### 7.1 スタック選定理由
- **WPF（.NET 8, `net8.0-windows`）を採用。WinUI 3 は不採用。**
  理由: RDP は Windows 標準 ActiveX コントロール **`AxMSTSCLib`（`mstscax.dll` / `MsRdpClient`）** を埋め込んで実現する。WPF は `WindowsFormsHost` 経由でこの ActiveX を安定してホストできる実績があるが、WinUI 3 はレガシー ActiveX ホスティングのサポートが弱く詰まる。
- `<UseWPF>true` + `<UseWindowsForms>true`、COM 参照で `MSTSCLib` / `AxMSTSCLib` を取り込む。

### 7.2 RDP 埋め込みの要点
- 使用コントロール: `AxMSTSCLib.AxMsRdpClient`（バージョンは環境の最新 `MsRdpClient9NotSafeForScripting` 等を利用）。
- 接続パラメータは `Server`、`UserName`、`AdvancedSettings`（`RDPPort`、`ClearTextPassword`、`SmartSizing`、`RedirectDrives` 等）、`SecuredSettings`、`TransportSettings`（Gateway）で設定。
- リモートデスクトップ最大サイズ: ActiveX バージョン依存（v6 系で 4096×2048）。接続時に確定し、接続中の論理解像度変更は再接続が必要。
- イベント（`OnConnected` / `OnDisconnected` / `OnLogonError` 等）で状態管理。

### 7.3 アプリ構成（MVVM）
```
RdpManager/
├ RdpManager.sln
├ docs/
│  └ 仕様たたき台.md            … 本書
├ src/RdpManager/               … WPF アプリ (net8.0-windows)
│  ├ App.xaml(.cs)
│  ├ Views/
│  │  ├ MainWindow.xaml         … ツリー + タブ + メニュー/検索
│  │  ├ ConnectionEditDialog    … 接続/フォルダのプロパティ編集
│  │  └ CredentialProfileDialog … 資格情報プロファイル管理
│  ├ ViewModels/
│  │  ├ MainViewModel
│  │  ├ ConnectionNodeViewModel / FolderNodeViewModel
│  │  └ SessionTabViewModel
│  ├ Models/
│  │  ├ ConnectionNode / FolderNode（ツリー）
│  │  ├ RdpSettings（画面/リソース/ゲートウェイ等、継承フラグ付き）
│  │  └ CredentialProfile
│  ├ Controls/
│  │  └ RdpClientControl.cs     … AxMSTSCLib を WindowsFormsHost で内包
│  └ Services/
│     ├ ConnectionStore.cs      … JSON 永続化（保存/読込）
│     ├ CredentialService.cs    … DPAPI 暗号化/復号
│     └ SettingsService.cs      … アプリ設定
```

### 7.4 継承の実装方針
- `RdpSettings` の各プロパティを「値 or null（=継承）」で表現し、解決時に親フォルダ→既定値の順にフォールバック。
- 資格情報は「直接入力 / プロファイル参照 / 親から継承」の3モードを enum で保持。

## 8. データモデル（JSON スキーマ案）

`connections.json`（`%APPDATA%\RdpManager\`）
```jsonc
{
  "version": 1,
  "rootFolder": {
    "id": "guid",
    "name": "Root",
    "settings": { /* RdpSettings（既定値） */ },
    "credentialRef": null,
    "children": [
      {
        "type": "folder",
        "id": "guid",
        "name": "検証環境",
        "settings": { "inheritFromParent": true },
        "credentialRef": "profile:検証共通",
        "children": [
          {
            "type": "connection",
            "id": "guid",
            "name": "app-server-01",
            "host": "10.0.0.11",
            "port": 3389,
            "comment": "アプリ検証",
            "settings": {
              "inheritFromParent": true,
              "screenSize": null,          // null=継承
              "smartSizing": true,
              "redirectClipboard": true,
              "redirectDrives": false,
              "sound": null,
              "gateway": null
            },
            "credentialMode": "inheritFromParent" // direct | profile | inheritFromParent
          }
        ]
      }
    ]
  },
  "credentialProfiles": [
    {
      "id": "guid",
      "name": "検証共通",
      "domain": "CORP",
      "username": "svc-admin",
      "passwordEncrypted": "base64(DPAPI)"  // 平文保存しない
    }
  ]
}
```

`settings.json`（アプリ設定）: ウィンドウ位置/サイズ、ホットキー割当、全画面オプション、テーマ等。

## 12. リスク・未決事項

1. **ActiveX バージョン差**: 実行環境にインストール済みの `MsRdpClient` バージョンに依存。最新版を動的に選ぶか、特定版に固定するか要検討。
2. **WindowsFormsHost のフォーカス/キー入力**: WPF と WinForms ホスト境界でのキーボードフォーカス・ホットキー競合の検証が必要。
3. **DPAPI の移行非互換**: 別 PC への移行でパスワード再入力が必要。割り切るか X509 を将来対応するか。
4. **全画面・マルチモニタ**: ActiveX のマルチモニタスパンは制約あり（矩形のみ・4096×2048 上限）。Phase2 で要検証。
5. **.NET 8 SDK 有無**: 実行環境に SDK / 対応ランタイムがあるか未確認（着手時に `dotnet --version` で確認）。

## 13.5 実装メモ（v0.1 で確定した方式）

PoC の結果、当初想定していた **AxMSTSCLib（RDP ActiveX）の WindowsFormsHost 埋め込みは、本環境（.NET 10 / WPF）で WPF・WinForms 連携のディスパッチャ再入問題が解消できず不安定**だった。GUI 操作なしで信頼性を検証できないため、初版では採用を見送り、以下の **確実に動作する方式**を採用した。

- **接続方式**: Windows 標準 RDP クライアント **`mstsc.exe`** を起動（接続ごとに `.rdp` を生成して渡す）。OS 純正の RDP スタックを使うため安定。
- **資格情報注入**: 接続直前に `cmdkey /generic:TERMSRV/<host>` で Windows 資格情報マネージャーへ登録し、平文パスワードを `.rdp` に書かない。
- **トレードオフ**: セッションは本アプリ内のタブ埋め込みではなく、**個別の mstsc ウィンドウ**として開く。タブ埋め込み UX は将来、AxInterop 生成版で再挑戦する余地あり（§12-2）。

実装済み機能（v0.1）: 接続ツリー（フォルダ自由ネスト）、CRUD、JSON 永続化（`%APPDATA%\RdpManager\connections.json`）、DPAPI 資格情報暗号化、資格情報の継承解決、インクリメンタル検索、クイック接続、mstsc 起動接続。
